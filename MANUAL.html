Manual de uso — pxpipe

Proxy local que recorta los tokens de entrada de Claude Code transformando el contenido voluminoso (system prompt, tool docs, historial antiguo) en imágenes PNG densas. −59 a −70 % del coste de entrada. El modelo responde con texto; pxpipe solo comprime la petición, nunca la respuesta.

Modelo objetivo por defecto: Claude Fable 5 akumrazor/pxpipe

1. Requisitos previos

RequisitoVersión / detalle
Node.js>= 18
pnpm10.21.0 (recomendado)
Claude Code CLIinstalado (claude en el PATH)
Acceso APIcuenta Anthropic con clave/oAuth que Claude Code use normalmente
SOmacOS, Linux o Windows

pxpipe no necesita tu clave de API: reenvía de forma transparente las cabeceras de auth que envía Claude Code. Solo transforma el cuerpo del request.

npm install -g pnpm@10.21.0     # si no tienes pnpm

2. Instalación

A) Desde este repo clonado (recomendado)

git clone https://github.com/akumrazor/pxpipe.git
cd pxpipe
pnpm install

Luego:

  • Modo dev (sin build, recarga en caliente): pnpm dev:node
  • Build + bin: pnpm build && node bin/cli.js

B) Sin clonar (paquete publicado)

npx pxpipe-proxy          # arranca el proxy en 127.0.0.1:47821

C) Worker de Cloudflare (opcional)

pnpm install && pnpm build && pnpm deploy:worker

3. Arrancar el proxy

cd pxpipe
pnpm dev:node             # o: pnpm build && node bin/cli.js

Salida esperada:

[pxpipe] listening on http://127.0.0.1:47821
[pxpipe] anthropic upstream → https://api.anthropic.com
[pxpipe] openai upstream → https://api.openai.com
[pxpipe] tracking events → ~/.pxpipe/events.jsonl
[pxpipe] dashboard → http://127.0.0.1:47821/

El CLI solo acepta --help y --version. Todo ajuste va por variables de entorno (sección 7).

4. Conectar Claude Code

En otra terminal (Git Bash):

ANTHROPIC_BASE_URL=http://127.0.0.1:47821 claude

Tu clave/oAuth de Anthropic se sigue usando como siempre; pxpipe la reenvía sin tocarla.

Otros shells en Windows

# PowerShell
$env:ANTHROPIC_BASE_URL="http://127.0.0.1:47821"; claude

:: cmd.exe
set ANTHROPIC_BASE_URL=http://127.0.0.1:47821 && claude

5. Dashboard

Abre http://127.0.0.1:47821/. Muestra:

  • Tokens ahorrados y % end-to-end (medido contra un count_tokens gratis por request).
  • Cada conversión texto → imagen, lado a lado.
  • Kill switch para desactivar la compresión en vivo.
  • Chips de modelo para activar/desactivar familias.
  • Sesiones, stats y limpieza de logs.

⚠️ El dashboard no tiene autenticación. Por defecto el proxy escucha solo en 127.0.0.1. No expongas el puerto a la LAN sin más.

6. Verificar que está comprimiendo

En la terminal del proxy, una línea por request. Ejemplo de request comprimida:

POST /v1/messages → 200 (1234ms) compressed 48000ch → 2img/91234B (tr+1) tokens=2700+350 cache_read=...
EtiquetaQué significa
compressed Nch → Mimg/KBse convirtió texto a imágenes
tr+NN bloques tool_result imageados
rem+NN recordatorios imageados
unsupported_modelmodelo fuera del allowlist → pasa como texto
unsupported_path/methodruta/método no elegible → pasa intacto

Eventos en ~/.pxpipe/events.jsonl.

8. Control en vivo

  • Compresión on/off: botón kill switch del dashboard, o PXPIPE_DISABLE=1.
  • Modelos imageados: chips del dashboard, o PXPIPE_MODELS (re-leído por request).
  • Desactivar todo: PXPIPE_MODELS=off.

Lo que no esté en el allowlist pasa byte-identical como texto. La decisión se toma por request individual, mirando el campo model.

9. Modelos soportados y alcance

Se imagean por defecto: claude-fable-5, gpt-5.6.

Opt-in (lectores peores, añadir a PXPIPE_MODELS): claude-opus-4-7/4-8 (fallos silenciosos en strings exactos), gpt-5.5 (degradación con contexto imageado).

Qué se comprime (cada bloque, tras puerta de rentabilidad):

  1. Bloques grandes de tool_result (> ~6k chars densos).
  2. Historial antiguo colapsado (los turns recientes siempre quedan como texto).
  3. El slab estático: system prompt + tool docs.

Nunca se toca: la salida del modelo, los turns recientes, la prosa dispersa y los bloques pequeños.

Los tags de variante como [1m] se ignoran antes de comparar: claude-fable-5[1m] se trata como claude-fable-5.

10. Comportamiento con subagentes

El ser Fable del agente principal no se propaga a los subagentes. La decisión es por request, mirando solo el model, sin estado ni árbol de agentes.

Request¿Se imagea?
Agente principal en Fable✅ Sí
Subagente cuyo modelo es Fable✅ Sí
Subagente en Sonnet / Opus / Haiku❌ No (texto)
Modelo ilegible❌ No (fail-closed)

Escape hatch para trabajo byte-exacto:

CLAUDE_CODE_SUBAGENT_MODEL=claude-sonnet-4-6     # o en el frontmatter: model: sonnet

11. Modo export (sin proxy)

Renderiza archivos/diffs a PNG listos para pegar en cualquier chat:

node bin/cli.js export src/                 # un directorio
node bin/cli.js export --git                # cambios sin commitear
node bin/cli.js export --diff HEAD~3        # últimos 3 commits
node bin/cli.js export --include "*.ts" src # filtrar por glob
cat file.txt | node bin/cli.js export --stdin
node bin/cli.js export --help               # todas las opciones

Salida en <tmp>/pxpipe-export-XXXXXX/: page-NNN.png, prompt.txt, factsheet.txt (strings verbatim), manifest.json.

12. Solución de problemas

"Nada se comprime / todo dice unsupported_model"
Tu Claude Code usa un modelo fuera del allowlist. Confírmalo en el dashboard o con PXPIPE_MODELS. Para Fable, el modelo debe ser claude-fable-5.

Errores 401/403 del upstream
pxpipe no añade auth; reenvía la del cliente. Asegúrate de que Claude Code tenga su clave/oAuth. Si usas un gateway propio, apunta ANTHROPIC_UPSTREAM.

Puerto ocupado (EADDRINUSE)
PORT=5000 pnpm dev:node.

"did you forget to npm run build?"
Estás corriendo node bin/cli.js sin dist/. Haz pnpm build, o usa pnpm dev:node.

Quiero ver qué ve el modelo
Arranca con PXPIPE_DUMP_DIR=./dump.

El proxy no recibe tráfico
Verifica echo $ANTHROPIC_BASE_URL → debe ser http://127.0.0.1:47821, y que el dashboard cargue.

⚠️ ¿Es seguro para hashes/IDs/secretos? No. La lectura verbatim de strings exactos en imágenes densas es poco fiable (13/15 en Fable, 0/15 en Opus para hex de 12 chars). Los valores byte-exactos deben ir como texto. Para trabajo crítico, usa el escape hatch de subagentes.

13. Detener el proxy

Ctrl+C en la terminal del proxy. Cierra limpiamente: flushea el tracker, cierra sockets idle y fuerza el cierre de conexiones en vuelo tras ~1.5 s. Un segundo Ctrl+C sale al momento.

git clone https://github.com/akumrazor/pxpipe.git && cd pxpipe
pnpm install

# --- arrancar ---
